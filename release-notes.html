    • Thanks to Norbert Vischer, if the 'type' argument of the Plot.add(type,xpoints,ypoints) macro function and plot.add(type,xpoints,ypoints) method starts with "code: " then the macro code contained in the 'type' string is executed per point, using variable names 'x' and 'y' for the point location and 's' for high-resolution scale factor. The Help>Examples>Custom Plot Symbols command provides an example.

    • Thanks to Michael Schmid, fixed a 1.51q reqression that could cause an exception when creating a plot.
